# Documentação De Segurança

Links principais: [README principal](../../README.md) | [Política inicial de segurança](../../SECURITY.md) | [Reporte de vulnerabilidades](vulnerability-reporting.md) | [Política de uso responsável](../legal/usage-policy.md) | [Guia de contribuição](../../CONTRIBUTING.md)

## Objetivo

Esta pasta reúne a documentação inicial de segurança do Vercosa AI Framework.

## Estado Atual

A documentação de segurança é inicial, conservadora e adequada ao estágio atual do projeto. O projeto ainda está em desenvolvimento, não possui release estável e ainda não publicou alfa pública.

Esta documentação será amadurecida conforme o projeto se aproximar de uma release pública, incluindo definição de canal público de reporte, processo de disclosure e nível de suporte aplicável.

## Documentos

- [Política inicial de segurança](../../SECURITY.md): descreve versões suportadas, limites atuais, práticas recomendadas e áreas de atenção.
- [Reporte de vulnerabilidades](vulnerability-reporting.md): orienta como preparar relatos responsáveis sem expor dados sensíveis.
- [Política inicial de uso responsável](../legal/usage-policy.md): descreve uso conservador, limites atuais e necessidade de revisão humana.
- [Guia de contribuição](../../CONTRIBUTING.md): descreve fluxo inicial de contribuição, validações e cuidados com secrets.

## Limites

Esta pasta não duplica toda a política de segurança, não cria SLA, não define bug bounty, não promete conformidade regulatória e não implementa mecanismos técnicos de segurança.

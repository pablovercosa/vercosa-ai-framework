# Visão do Vercosa AI Framework

O Vercosa AI Framework é um framework open source para desenvolvimento de software orientado por especificações e assistido por inteligência artificial.

Ele nasce da necessidade de organizar o uso de IA no desenvolvimento de software de forma disciplinada, rastreável, extensível, segura e econômica.

A proposta não é apenas gerar código com IA.

A proposta é criar um processo completo de engenharia de software onde ideias se transformam em especificações, arquiteturas, tarefas, código, testes, documentação, conhecimento e aprendizado contínuo.

## Problema

As ferramentas atuais de IA para desenvolvimento costumam operar de forma centrada em prompt.

O usuário pede algo.
O modelo responde.
Às vezes gera código.
Às vezes erra.
Às vezes acerta.
O processo é pouco rastreável.

Faltam:

- especificações como fonte da verdade;
- governança;
- rastreabilidade;
- economia de tokens;
- segurança;
- controle de qualidade;
- escolha inteligente de modelos;
- reutilização de conhecimento;
- separação entre agentes, ferramentas e infraestrutura;
- arquitetura agnóstica de fornecedor.

## Proposta

O Vercosa AI Framework organiza o desenvolvimento em missões.

Cada missão passa por:

1. Specification
2. Architecture
3. ADR
4. Planning
5. Implementation
6. Review
7. Testing
8. Documentation
9. Indexing
10. Continuous Learning

## Diferencial

O framework não é orientado a prompts.

É orientado a:

- missões;
- specs;
- policies;
- workflows;
- agents;
- capabilities;
- loops;
- knowledge;
- governance.

## Não objetivos

O framework não deve ser:

- dependente de OpenCode;
- dependente de Claude;
- dependente de PostgreSQL;
- dependente de Ollama;
- dependente de ARM64;
- dependente de uma única IDE;
- dependente de um único banco vetorial;
- dependente de um único provedor de IA.

Essas ferramentas podem ser adapters, nunca o núcleo.
